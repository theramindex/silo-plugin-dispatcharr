# Dispatcharr Live TV

This context describes the Live TV experience powered by Dispatcharr inside Silo. It defines the product language for browsing, discovering, watching, and recording live television.

## Language

**Search**:
A discovery surface that finds watchable or recordable Live TV items across channels, groups, guide programs, sports, events, and recordings.
_Avoid_: Row filter, guide filter

**On Later**:
A browse surface for upcoming guide content organized by useful viewing categories such as movies, sports, events, shows, live, and today.
_Avoid_: Future EPG dump

**Pass**:
A user-profile saved rule that tracks matching guide content for later viewing or recording.
_Avoid_: Alert, playlist

**Team Pass**:
A **Pass** based on a favorite sports team.
_Avoid_: Team favorite when referring to the saved discovery rule

**Keyword Pass**:
A **Pass** based on guide text matching a user-supplied keyword or phrase.
_Avoid_: Search history

**Series Pass**:
A future **Pass** based on repeated airings of the same show-like guide title.
_Avoid_: Series when metadata is not strong enough to identify an actual show

**Collection**:
A channel grouping surface, including source groups, featured groups, virtual groups, and user custom groups.
_Avoid_: Category when speaking to users

**Channel Flipping**:
A player-side discovery pattern that shows what is on nearby or related channels while watching.
_Avoid_: Search

**Multiview**:
A watch mode that plays multiple channels at once with one active audio source.
_Avoid_: Picture-in-picture when more than one extra stream is involved

**Rewind Channel**:
A channel whose stream supports pause, seek, or rewind behavior.
_Avoid_: DVR recording

**Recording Availability**:
Whether the configured Dispatcharr source can technically schedule recordings.
_Avoid_: Recording permission

**Recording Default**:
An admin setting that decides whether recording UI is enabled by default when **Recording Availability** exists.
_Avoid_: Security gate

## Relationships

- **Search** may return **Collections**, channels, guide programs, sports events, broadcast events, and recordings.
- **On Later** is built from guide programs and may surface items that also match a **Pass**.
- A **Pass** belongs to the current Silo user profile.
- A **Team Pass** can use sports data and guide text to find related programs.
- A **Collection** contains channels and can be generated by Dispatcharr group names, Silo virtual groups, featured groups, or user custom groups.
- **Recording Availability** depends on Dispatcharr Direct mode.
- **Recording Default** affects UI behavior only and is not a security boundary.
- **Rewind Channel** behavior depends on stream/source support, not on scheduled recordings.

## Example dialogue

> **Dev:** "Should Search only search the guide?"
> **Domain expert:** "No. **Search** should find things to watch or record. The guide powers program results, but channels and **Collections** still need to work when guide data is thin."

> **Dev:** "Can we call a repeated title a Series?"
> **Domain expert:** "Not yet. Until metadata is stronger, call it upcoming airings and reserve **Series Pass** for later."

## Flagged ambiguities

- "Category" previously referred to Dispatcharr groups, virtual folders, featured rows, and user custom groups. User-facing language is now **Collection** or **Group** depending on surface; code may still use category internally.
- "Can record" can mean source support, default UI behavior, or secure user entitlement. For now, implement **Recording Availability** and **Recording Default** only; do not claim a security gate.
